# План проверки MVP

## Smoke

```bash
docker compose up --build
./scripts/smoke.sh
```

Ожидаемый результат:

- контейнеры `oracle` и `web` запущены;
- web подключается к Oracle;
- `/login` отвечает без 500.

## Авторизация

- Вход `admin` / `password` успешен.
- Неверный пароль возвращает сообщение об ошибке.
- Logout завершает сессию.
- Открытие `/dashboard` без сессии перенаправляет на `/login`.

## CRUD

Для разделов `tools`, `staff`, `elements`, `documents`, `devices`:

- создать запись;
- увидеть запись в таблице;
- изменить запись;
- удалить запись;
- проверить, что обязательные поля валидируются.

## Импорт

Для разделов `tools`, `elements`, `devices`:

- кнопка `Создать` открывает окно выбора способа создания;
- `Вручную` открывает старую форму создания;
- загрузка `.txt` отправляет файл с CSRF-токеном;
- загрузка `.xlsx` отправляет файл с CSRF-токеном.

TXT:

- импортировать валидный файл оборудования с русскими заголовками и разделителем `;`;
- импортировать валидный файл элементов с системными заголовками;
- проверить ошибку при пустом обязательном поле;
- проверить ошибку при дубле уникального поля внутри файла;
- проверить ошибку при дубле уникального поля, уже существующего в БД.

XLSX:

- импортировать валидный первый лист;
- проверить русские подписи значений для select-полей;
- проверить системные значения для select-полей;
- проверить, что пустые строки пропускаются.

Для `staff` и `documents` кнопка `Создать` остается прямой ссылкой на форму.

## Роли

- `admin` видит все разделы.
- `engineer` видит только оборудование и изделия.
- `technologist` видит только элементы.
- `manager` видит только персонал.
- `director` видит только документы.
- Прямой переход на чужой раздел возвращает 403.

## PDF

- PDF-отчет формируется из каждого списка.
- PDF отдельного документа формируется из раздела документов.
